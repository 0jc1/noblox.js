    Introduction

    Welcome to the noblox.js API! This site will help you get started with noblox.js. Once you're familiar, you should be fine to navigate this website yourself.

    Authentication

    To log into a Roblox account, use this code:

    const noblox = require('noblox.js');

async function run() {
  await noblox.cookieLogin('_ROBLOSECURITYKEYFORCOOLBOY57');
}

run();
    The following code utilizes the async/await feature in ES8. Be sure you're running the latest node version and latest version of noblox.js.

    Make sure to replace _ROBLOSECURITYKEYFORCOOLBOY57 with your .ROBLOSECURITY key. You can find this by heading over to a Roblox page and signing in. Once you're in, open Inspect Element in your browser (Control + Shift + I). Click the Application Tab and find the .ROBLOSECURITY line and copy its contents, starting with _|WARNING:-DO. Put the full token, including the warning, into the cookieLogin line.

    Following Roblox's recent Funcaptcha update, users can no longer log into their accounts through noblox.js using the basic username/password combination. In order to get around this, noblox.js bypasses this and uses your .ROBLOSECURITY key to login. Your .ROBLOSECURITY key doesn't go anywhere apart from your local computer and to Roblox directly so you can trust the process.

    User Functions

    Accept Friend Request

    This function accepts an existing friend request.

    const nbx = require('noblox.js')

...
nbx.acceptFriendRequest(44545891)
...

    The method above returns a Promise that resolves if the friend request has been accepted successfully.

    Parameters

    Parameter Default Description Required
    userID none This is the user ID of the user that sent the friend request being accepted. ✓

    Cookie Login

    This function logs into Roblox using the given .ROBLOSECURITY cookie.

    const nbx = require('noblox.js')

...
nbx.cookieLogin('.ROBLOSECURITY')
...

    The method above returns a Promise containing the logged in user's information formatted as follows:

  UserName: 'Qxest',

    Parameters

    Parameter Default Description Required
    cookie none This is the .ROBLOSECURITY cookie of the account being logged into. ✓

    Get Blurb

    This function returns the blurb of a user. The blurb is the equivalent of the user's description on Roblox, not their status message.

    const noblox = require('noblox.js');

...
noblox.getBlurb(1)
...

    The method above returns a Promise that resolves with a string, in this case:

    "Welcome to the Roblox profile! This is where you can check out the newest items in the catalog,and get a jumpstart on exploring and building on our Imagination Platform. If you want news on updates to the Roblox platform, or great new experiences to play with friends, check out blog.roblox.com. Please note, this is an automated account. If you need to reach Roblox for any customer service needs find help at www.roblox.com/help"

    Parameters

    Parameter Default Description Required
    userID none This is the user ID of the user for which a blurb is returned. ✓

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Roblox API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
